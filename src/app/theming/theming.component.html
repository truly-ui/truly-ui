Truly-UI allows you to customize some basic design aspects in order to meet the needs of UI diversity from business and brand, including primary color, contrast colors, border color, etc..
A theme is the set of colors that will be applied to the TrulyUI components. In TrulyUI, a theme is created by composing multiple palettes. In particular, a theme consists of:
  • A basic palette: colors used for element backgrounds.
  • A primary palette: colors most widely used across all screens and components.
  • A success palette: colors used to convey success state.
  • A danger palette: colors used to convey danger state.
  • A warning palette: colors used to convey warning state.
  • A information palette: colors used to convey information state.
Truly-UI already counts a prepackaged package of several themes to suit any application. To use them simply specify in the creation of the TrulyUI Core Module CoreModule.forRoot({{ '{' }} theme: 'default' {{ '}' }}) which theme you want to use.

When you want more customization than a pre-built theme offers, you can create your own theme file.
TrulyUI also allows you to perform the Runtime swapping of the themes you created. Just specify them in the setTheme () function of the CoreService service.

We recommend that you use Theme switching in Runtime when you create multiple themes. So your user can switch between them.

A custom theme file does three things:
  • It does import builder that will perform the mixin of the color palettes @import "~truly-ui/css/theming/builders/basic/index";

  • Defines a theme data structure as the composition of multiple palettes. The Palette receives the name and its context variables (default, lighter, darker) in number(View the available palettes).

  • The object sass is passed to the truly-basic-builder-theme mixin, along with the name that will be the representation of the theme in CoreModule, which will output all of the corresponding styles for the theme.
